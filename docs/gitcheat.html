CADE Lab git Cheat Sheet

Lucas Adam M. Paul

1 Introduce yourself to git 

git config --global user.name "Luke Skywalker"
git config --global user.email skywalker@tatooine.nrp

2 A place to put shared repos 

mkdir -p ~/Public/cs3505/hw1 # Makes the project directory too
chmod -R 755 ~/Public # Set permissions for public readability
Only do that chmod once. It can screw things up if Public already exists and has stuff in it. If you need its effect and there’s already stuff in Public, use this alternative: 
find ~/Public -type d -exec chmod 755 {} \;

3 Create shared repo 

cd ~/Public/cs3505/hw1 # cd into public project dir
git init --bare # Create shared repo
git config core.sharedrepository 0644 # Set permissions

4 Cloning

Do this when you want a local repo (a “working copy”). 
cd ~/Documents/classes/cs3505 # or wherever you want the local
git clone /home/skywalke/Public/cs3505/hw1 # Within-CADE version
### OR ###
git clone ssh://skywalke@lab1-1.eng.utah.edu/home/skywalke/Public/cs3505/hw1 # Remote version
Replace skywalke with your username, lab1-1 with any lab machine, and the paths with whatever sensible paths are appropriate for your situation. Use the remote version from any non-CADE machine.

5 Committing

Commit things to your local repo. 
git add newfile.txt src/new.C # Stage new files for commit
git commit -a # Commit staged and modified files
An editor will pop up. Enter a commit message, save, and exit the editor. The commit will be saved.

6 Pushing

Make changes available to team members by pushing them. From the local repo directory: 
git push origin master
git push # Shortcut.
The shortcut version may not work the first time, but should work on subsequent pushes if you followed this guide.

7 Pulling

Get changes from your teammates by pulling.

7.1 Remote management

Remotes are nicknames for urls from which to pull. From the local repo directory: 
git remote add han /home/hsolo/Public/cs3505/hw1 # Within-CADE version
### OR ###
git remote add han ssh://skywalke@lab1-1.eng.utah.edu/home/hsolo/Public/cs3505/hw1 # Remote version
A remote’s name can be pretty much anything. If you need to change a remote’s url, do the above command, but with set-url instead of add.

7.2 Typical Pull

From the local repo directory: 
git pull han master # master is the branch
